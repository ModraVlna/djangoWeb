Nasazení aplikace

Aby vytvořená aplikace byla dostupná světu musí se zvolit verzovací systém a web hostingová služba.

Git

Pomocí Gitu lze ukládat kód v průběhu vývoje. Výhodou je, že kód je zaholován a lze si zobrazovat starší verze aplikace.
V této ukázce bude vybrán Github. Registrace a používaní Github je zdarma.
Po založení účtu na Githubu je potřeba provést určitá nastavení na svém zařízení.

Otevřeme terminál a zapneme virtuální prostředí pro náš projekt. Uvnitř adresáře, kde máme zdrojové kódy provedeme postupně 5 příkazů.

git config --global user.name "Vase jmeno"
git config --global user.email "muj@email.cz"
git config --global push.default matching
git config --global alias.co checkout
git init


Nyní máme na svém zařízení repositář, kam budeme vkládat náš kód, který chceme mít uložený na Github.
První příkaz přidá všechny soubory do našeho lokálního repositáře.

github add .


Kód jsme přidali do repositáře a pomocí následujícího příkazu ho uložíme.

git commit -am 'Prvni commit'


Zpráva v uvozovkách je informace o tom, jaké změny jsme provedli. Máme připravený kód, který bude vidět na Github po provedení dalších nastavení.

V novém terminále vytvoříme adresář, kam budeme ukládat SSH klíč pro zajištění bezpečné komunikace.

mkdir ~/.ssh


Vytvoříme SSH klíč.

ssh-keygen.exe


Vytvoří se public a private RSA key pair. Nás bude zajímat public klíč. Tento klíč musíme vložit do aplikace Github. Zobrazíme si tento klíč a zkopírujeme ho.

cat id_rsa.pub


Na Github v položce nastavení, v sekci SSH a GPG klíče přidáme nový SSH klíč. Do položky vložíme daný klíč.
Na Github vytvoříme repositář, který libovolně pojmenujeme.
Zobrazí se nám, že máme vložit dva příkazy. Tyto příkazy zkopírujeme z Github.
Otevřeme nás terminál a zapneme virtuální prostředí.
Příkazy mají tuto strukturu:

git remote add origin git@github.com:nazev/nazev.git
git push -u origin master


Poté v Github uvidíme repositář, kde je náš zdrojový kód.
Když provedeme nějaké změny v aplikace, tak použijeme pouze tři příkazy pro nahrání na Github.

git add .
git commit -am 'Pridana sablona'
git push


Heroku

Aplikace bude nasazena pomocí Heroku. Na Heroku vytvoříme zdarma účet a stáhneme program, který se nainstaluje.
Zavřeme na našem zařízení terminál s virtuálním prostředí a otevřeme nový terminál, kde spustíme virtuální prostředí.
Pro kontrolu, že je Heroku nainstalován na našem zařízení, provedeme příkaz, který vypíše danou verzi.

heroku --version


Pro práci Heroku a Djnago budeme potřebovat stáhnout balíčky.

pip install gunicorn
pip install django-heroku
pip install python-decouple


Vytvoříme soubor s názvem Procfile, kam napíšeme o jakou aplikaci se jedná.

touch Procfile


web: gunicorn nazev_aplikace.wsgi


Dále vytvoříme textový soubor, do kterého napíšeme verze balíčků.

pip freeze > requirements.txt


Nyní otevřeme soubor settings.py, ve kterém provedeme změny.

import os
import django_heroku
import dj_database_url
from decouple import config


Uvnitř MIDDLEWARE přidáme

'whitenoise.middleware.WhiteNoiseMiddleware',


Na konci settings.py přidáme

STATICFILES_STORAGE = 'whitenoise.storage.CompressedManifestStaticFilesStorage'
django_heroku(settings(locals))


Tyto změny musíme uložit a nahrát na Github.
Po nahraní se v terminálu přihlásíme do Heroku.

heroku login


Přidáme SSH klíče

heroku keys:add


Vytvoříme Heroku aplikaci

heroku create


Přejmenujeme si název adresy např na

heroku rename prvniaplikace


Pokud název adresa již existuje, musíte zvolit jiný název.
Nyní je adresa aplikace prvniaplikace.herokuapp.com

V terminálu provedeme příkaz

heroku config:set DISABLE_COLLECTSTATIC=1


Nahrajeme na Heroku náš kód z Github

git push heroku master


Kdyby příkaz nefungoval, lze použít

git push heroku main


Poté je webová aplikace dostupná světu.
